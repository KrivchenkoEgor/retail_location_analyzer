# Retail Location Analyzer

Streamlit-приложение для анализа торговых точек. Геокодинг, поиск зданий и квартир в зоне охвата, анализ конкурентов и социальной инфраструктуры, демографический расчёт, обогащение данными управляющих компаний, экспорт в Excel.

## Возможности

- **Геокодинг** — Почта России + Nominatim (резерв)
- **Дома и квартиры** — in-circle API Почты России, квартиры → жители (настраиваемая модель домохозяйств)
- **Конкуренты** — поиск продуктовых магазинов через OpenStreetMap (Overpass API) с группировкой по брендам
- **Офисы и образование** — бизнес-центры, ВУЗы, колледжи, ПТУ (OSM)
- **Обогащение** — данные из my-gkh.ru (год постройки, этажность, материал стен, УК, поставщики ресурсов)
- **Экспорт** — Excel-отчёт (7 листов: сводка, демография, детально, выводы, матрица сетей, офисы/образование, данные МКД)

## Установка и запуск

```bash
git clone https://github.com/KrivchenkoEgor/retail_location_analyzer.git
cd retail_location_analyzer
pip install -r requirements.txt
streamlit run app.py
```

Откроется в браузере по адресу `http://localhost:8501`.

## Использование

1. **Боковая панель**: укажите город, адреса (по одному в строке или загрузите `.txt`), радиусы поиска
2. **Демография**: отредактируйте модель распределения домохозяйств
3. **Запустите анализ** — прогресс-бар покажет статус каждой точки
4. **Результаты** — вкладки: Сводка, Детально, Демография, Матрица сетей, Данные МКД, Экспорт
5. **Скачайте Excel-отчёт** на вкладке «Экспорт» (7 листов)

## Источники данных

| Данные | Источник |
|--------|----------|
| Геокодинг | geo.pochta.ru + nominatim.openstreetmap.org |

## Демографическая модель

- Базовая оценка: `квартиры × 2,3`
- Скорректированная: `квартиры × 2,44` (поправка на долю одиноких 28,5%)
- Распределение домохозяйств: 1 чел — 23,5% / 2 — 30% / 3 — 23% / 4 — 15% / 5+ — 8,5%
- Все параметры редактируются в UI

## Замечания

- Overpass API медленный (~10 с на точку); между точками пауза для защиты от блокировки
- `flats = 1` трактуется как «нет данных по жилому фонду» (аналогично фронтенду Почты России)
- my-gkh.ru может требовать сессию браузера (CAPTCHA) при запросах к `/gethouse/` — модуль пытается получить данные напрямую; если не удаётся, детальная информация пропускается, базовые данные (УК, адрес) доступны всегда
